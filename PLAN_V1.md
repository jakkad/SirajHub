# SirajHub — Personal Content Tracking Hub — Plan V1

## Context
Building a personal media/content tracker for Books, Movies, TV Shows, Podcasts, YouTube, Articles, and Tweets. Items tracked across four statuses: Suggestions, In Progress, Finished, Archived. AI auto-categorizes new items and optionally generates summaries + a ranked "next to consume" list. Runs free on Cloudflare Workers (shared hosting not used — Workers replaces it entirely).

## Architecture

```
Browser (React SPA — served from Workers Static Assets)
    │  Hono RPC types (end-to-end type-safe)
    ▼
Cloudflare Worker (Hono v4)
 ├── /api/items       CRUD operations
 ├── /api/ingest      URL → metadata fetcher dispatcher
 ├── /api/ai          summarize / rank / categorize
 └── /auth/*          Better Auth handler
    │
    ├── D1 Database (Drizzle)
    ├── KV Store (sessions + URL cache)
    └── Gemini API (outbound fetch — no CPU cost)
            │
            └── External Metadata APIs
                 Books: Open Library + Google Books
                 Movies/TV: TMDB (free key)
                 Podcasts: Podcast Index + RSS parsing
                 YouTube: YouTube Data API v3
                 Articles: OG tag scraping (self-hosted)
                 Tweets: Twitter oEmbed (no key)
```

## Database Schema (D1 / SQLite via Drizzle)

**Tables:**
- `users` — id (ulid), email, name, preferences (JSON: taste profile for AI)
- `sessions` — Better Auth managed
- `items` — id, user_id, content_type, status, source_url, external_id, title, creator, description, cover_url, release_date, duration_mins, metadata (JSON blob for type-specific extras), position (drag order), rating, notes, started_at, finished_at
- `tags` + `item_tags` — many-to-many, user-scoped
- `ai_cache` — content_id, analysis_type, model_used, prompt_hash, result (JSON), created_at
- `url_cache` — url (PK), metadata (JSON), fetched_at, source

**Design decisions:**
- `metadata TEXT` JSON blob on items (avoids 7 type-specific tables; fast enough for personal ~500-item scale)
- ULIDs throughout (sortable, URL-safe)
- Unix timestamps (integers) for all dates
- `position INTEGER` enables drag-and-drop reordering without touching every row

## AI Pipeline

### Auto-categorize on add (every new item)
Lightweight call to Gemini Flash-Lite:
```
Input: title + description + url domain
Output JSON: { content_type, confidence, suggested_tags, suggested_status }
```

### On-demand analysis (user clicks "Analyze")
1. Check D1 `ai_cache` → return if < 7 days old
2. Build prompt by content type (book → bullets+rating, article → TL;DR, movie → digest+mood)
3. Gemini Flash-Lite call with structured JSON output schema
4. Store in `ai_cache`, return to client

### "Next to Watch/Read" ranked list
1. Pull all `status = 'suggestions'` items from D1
2. Bundle into single prompt with user's taste preferences
3. Gemini ranks + gives 1-line reasoning per item
4. Cache result in KV for 6 hours

**Free tier budget:** Gemini 1k req/day vs. ~20–30 personal daily calls. Never hits the limit.

- **Mobile:** single-column list; Board collapses to horizontal scroll

## Auth

- Better Auth with email+password (personal use = one user, register once)
- Sessions in D1 (persistent) + KV (fast lookup)
- HTTP-only cookie, 30-day sliding window
- Hono middleware injects `user_id` into context before every `/api/*` route
- **Public launch path:** enable registration + Google OAuth in Better Auth config (one-line each), all queries already scoped by `user_id`

5. **AI features** — Gemini integration, auto-categorize on add, manual summarize, "next list"
6. **Polish** — Grid view, tags, search/filter, settings page, responsive mobile
